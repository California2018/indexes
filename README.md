### Задание 3
Это отличное задание для понимания того, почему PostgreSQL называют «самой продвинутой объектно-реляционной базой данных».

В то время как MySQL (с движком InnoDB) в основном полагается на B-Tree (стандартный индекс) и R-Tree (для геометрии), а также Full-text (для поиска), PostgreSQL предлагает гораздо более широкий набор специализированных индексов для сложных типов данных.

Вот список типов индексов, которые есть в PostgreSQL, но отсутствуют в MySQL (или реализованы принципиально иначе/хуже):

1. GIN (Generalized Inverted Index)
Это «Обобщенный обратный индекс».

Зачем нужен: Идеален для типов данных, которые содержат множество значений в одной ячейке. Это прежде всего массивы (Arrays) и JSONB.
Отличие от MySQL: В MySQL для индексации JSON обычно приходится создавать "виртуальные колонки" (Generated Columns) и вешать на них обычный B-Tree индекс. В PostgreSQL вы вешаете GIN-индекс прямо на весь JSON-документ, и он эффективно ищет вхождения ключей и значений внутри него. Также GIN используется для полнотекстового поиска (tsvector), работая быстрее и гибче аналога в MySQL.
2. GiST (Generalized Search Tree)
Это «Обобщенное дерево поиска».

Зачем нужен: Это даже не один индекс, а инфраструктура для создания индексов под сложные, нестандартные типы данных. На его основе реализованы индексы для геометрических фигур, сетевых адресов и диапазонов.
Отличие от MySQL: В MySQL есть R-Tree для геометрии (Spatial), но GiST в Postgres гораздо мощнее. Например, он позволяет индексировать диапазоны (Range types) — временные интервалы или числовые отрезки, позволяя мгновенно находить пересечения отрезков (&&). MySQL не умеет эффективно индексировать пересечения диапазонов.
3. SP-GiST (Space-Partitioned GiST)
Это «GiST с пространственным разбиением».

Зачем нужен: Используется для данных, которые неравномерно распределены (кластеризованы) или имеют естественную иерархию.
Примеры: Подходит для индексации URL-адресов, путей к файлам, IP-адресов. Реализует такие структуры данных, как деревья квадрантов (quadtrees), k-d деревья и префиксные деревья (radix trees). В MySQL аналогов нет.
4. BRIN (Block Range INdex)
Это «Индекс зон/диапазонов блоков».
